# 밥줘 🍚 - AI 기반 학생식당 스마트 주문 서비스

## 1. 프로젝트 개요

**밥줘**는 강원대학교 삼척캠퍼스 학생식당의 점심시간 혼잡 문제를 해결하기 위한 AI 기반 스마트 주문 웹앱입니다.

학생은 앱에서 실시간 혼잡도, AI 추천 방문시간, 메뉴 주문, 픽업 상태를 확인할 수 있고, 식당 운영자는 관리자 대시보드에서 주문 현황, 메뉴 판매 순위, 다음달 식수 예측과 운영 추천을 확인할 수 있습니다.

---

## 2. 문제 정의

학생식당은 점심시간에 이용자가 특정 시간대에 몰리면서 다음 문제가 발생합니다.

- 키오스크 앞 긴 대기줄 발생
- 제한된 점심시간 안에 식사 시간이 부족해짐
- 식당 내부 혼잡도 증가
- 운영자는 시간대별 수요를 정확히 파악하기 어려움
- 식자재 준비량 예측이 어려워 잔반과 재고 낭비가 발생할 수 있음

---

## 3. 핵심 해결 방식

밥줘는 다음 기능을 통해 문제를 해결합니다.

1. **AI 실시간 혼잡도 예측**  
   현재 시간, 운영시간, 과거 판매 데이터, 학사일정, 날씨 데이터를 고려해 혼잡도를 예측합니다.

2. **AI 추천 방문시간**  
   혼잡도가 높은 점심 피크 시간대를 피하고 비교적 여유로운 방문 시간대를 추천합니다.

3. **모바일 사전 주문**  
   학생이 앱에서 메뉴를 미리 담고 주문할 수 있도록 구성했습니다.

4. **픽업 상태 관리**  
   주문 상태를 `접수완료 → 조리중 → 픽업완료` 단계로 보여줍니다.

5. **관리자 대시보드**  
   현재 주문, 메뉴 판매 순위, AI 혼잡도, 다음달 식수 예측을 확인할 수 있습니다.

6. **개발자 시연 모드**  
   공모전 발표 시 운영시간이 아니더라도 `여유`, `보통`, `혼잡`, `매우붐빔` 상태를 즉시 시연할 수 있습니다.

## 5. AI 혼잡도 계산 구조

AI 혼잡도는 다음 요소를 종합해 계산합니다.

```text
현재 시간
+ 운영시간 여부
+ 과거 시간대별 판매 패턴
+ 11~12시 점심 피크 가중치
+ 시험기간 여부
+ 방학 여부
+ 날씨 영향
+ 최근 주문량 시뮬레이션
= 혼잡도 점수(0~100)
```

혼잡도 점수 기준은 다음과 같습니다.

| 점수 | 상태 |
|---:|---|
| 0~25 | 여유 |
| 26~50 | 보통 |
| 51~75 | 혼잡 |
| 76~100 | 매우붐빔 |

## 7. Firebase / OpenAI 연동 구조

실제 운영 환경에서는 OpenAI API Key를 브라우저 코드에 직접 넣지 않습니다.

구조는 다음과 같습니다.

```text
Browser(index.html)
↓
Firebase Hosting
↓
Firebase Functions
↓
OpenAI Responses API
↓
JSON 반환
↓
HTML 화면 렌더링
```

### 보안 설계

- OpenAI API Key는 브라우저에 노출하지 않습니다.
- API Key는 Firebase Functions 환경변수에 저장합니다.
- 브라우저는 Firebase Functions의 `/predictCongestion` API만 호출합니다.
