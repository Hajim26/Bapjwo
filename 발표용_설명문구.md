# 밥줘 발표용 설명 문구

## 1분 요약 버전

안녕하세요. 저희 팀은 강원대학교 삼척캠퍼스 학생식당의 점심시간 혼잡 문제를 해결하기 위해 **AI 기반 학생식당 스마트 주문 서비스 ‘밥줘’**를 기획하고 구현했습니다.

현재 학생식당은 점심시간에 학생들이 한꺼번에 몰리면서 키오스크 대기줄, 식사 시간 부족, 식당 내부 혼잡 문제가 반복되고 있습니다. 밥줘는 이러한 문제를 해결하기 위해 과거 판매 데이터, 운영시간, 학사일정, 날씨 데이터를 기반으로 AI가 실시간 혼잡도와 추천 방문시간을 제공합니다.

학생은 앱에서 혼잡도를 확인하고, 덮밥·백반·1000원의 아침밥을 미리 주문한 뒤, 픽업 상태를 확인할 수 있습니다. 운영자는 관리자 대시보드를 통해 현재 주문 현황, 메뉴 판매 순위, 다음달 예상 식수와 준비량 추천을 확인할 수 있습니다.

특히 OpenAI API는 브라우저에 직접 연결하지 않고 Firebase Functions를 통해 호출하도록 설계하여 보안성을 고려했습니다. 또한 개발자 시연 모드를 통해 실제 운영시간이 아니더라도 여유, 혼잡, 매우붐빔 상태를 즉시 시연할 수 있도록 구현했습니다.

밥줘는 단순 주문 앱이 아니라, 학생의 대기시간을 줄이고 식당 운영자의 수요 예측과 식자재 준비량 최적화까지 지원하는 확장형 AI 운영 서비스입니다.

## 3분 발표 버전

저희가 해결하고자 한 문제는 강원대학교 삼척캠퍼스 학생식당의 점심시간 혼잡입니다. 점심시간에는 학생들이 특정 시간대에 한꺼번에 몰리면서 키오스크 앞 대기줄이 길어지고, 식사를 위해 기다리는 시간이 늘어나며, 식당 내부도 혼잡해지는 문제가 발생합니다.

이를 해결하기 위해 저희는 **AI 기반 학생식당 스마트 주문 서비스 ‘밥줘’**를 만들었습니다.

밥줘의 첫 번째 핵심 기능은 **AI 실시간 혼잡도 예측**입니다. 앱은 현재 시간, 운영시간, 과거 키오스크 판매 데이터, 학사일정, 시험기간 여부, 방학 여부, 날씨 데이터를 종합해 현재 혼잡도 점수를 계산합니다. 혼잡도는 0부터 100까지 점수로 표현되며, 여유·보통·혼잡·매우붐빔 단계로 사용자에게 보여줍니다.

두 번째 기능은 **AI 추천 방문시간**입니다. 과거 판매 데이터에서 11시부터 12시 사이에 주문이 집중되는 패턴을 반영하여, 학생에게 상대적으로 덜 붐비는 시간대를 추천합니다. 추천 결과는 신뢰도 퍼센트바와 함께 표시되어 사용자가 쉽게 판단할 수 있도록 했습니다.

세 번째 기능은 **모바일 사전 주문과 픽업 관리**입니다. 학생은 덮밥, 백반, 1000원의 아침밥 중 메뉴를 선택하고 주문할 수 있습니다. 주문 후에는 접수완료, 조리중, 픽업완료 상태가 단계별로 표시되어 식당 앞에서 불필요하게 오래 기다리지 않아도 됩니다.

네 번째는 **관리자 대시보드**입니다. 운영자는 오늘 주문 수, 픽업 대기인원, 예상 매출, 메뉴 판매 순위, 현재 주문 큐를 확인할 수 있습니다. 또한 AI 다음달 식수 예측 기능을 통해 다음달 예상 총 식수, 일평균 식수, 피크 시간, 메뉴별 준비량 추천을 확인할 수 있습니다. 이를 통해 식자재 준비량을 최적화하고 잔반과 재고 낭비를 줄일 수 있습니다.

기술 구조 측면에서는 보안도 고려했습니다. OpenAI API Key를 브라우저 코드에 직접 넣지 않고, Firebase Functions에서 OpenAI API를 호출하는 구조를 설계했습니다. 브라우저는 Firebase Functions로 요청을 보내고, 서버가 OpenAI 응답을 JSON 형식으로 받아 다시 앱에 전달합니다.

마지막으로 공모전 시연을 위해 개발자 시연 모드를 추가했습니다. 실제 운영시간이 아니어도 여유, 보통, 혼잡, 매우붐빔 상태를 버튼으로 즉시 전환하여 AI 예측 화면을 시연할 수 있습니다.

결론적으로 밥줘는 학생에게는 대기시간 감소와 편리한 주문 경험을 제공하고, 식당 운영자에게는 수요 예측과 운영 최적화를 지원하는 AI 기반 스마트 학식 플랫폼입니다.

- “단순히 현재 혼잡도를 보여주는 것이 아니라, 과거 판매 데이터와 학사일정을 기반으로 앞으로의 혼잡을 예측합니다.”
- “학생용 앱과 관리자용 대시보드를 함께 구성해 실제 운영 가능성을 높였습니다.”
- “OpenAI API Key를 브라우저에 노출하지 않고 Firebase Functions에서만 호출하도록 설계했습니다.”

- “개발자 시연 모드로 운영시간이 아니어도 다양한 혼잡 상황을 즉시 보여줄 수 있습니다.”
