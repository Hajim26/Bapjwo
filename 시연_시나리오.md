# 밥줘 시연 시나리오

## 시연 목표

심사위원에게 밥줘가 단순한 화면 디자인이 아니라, 학생 주문 흐름과 식당 운영 최적화까지 고려한 AI 서비스라는 점을 보여준다.

---

## 권장 시연 순서

### 1. 홈 화면 소개

보여줄 내용:

- AI 실시간 혼잡도
- AI 추천 방문시간
- 신뢰도 퍼센트바
- 픽업 대기인원
- 예상 잔여 좌석

설명 예시:

> 홈 화면에서는 학생이 식당에 가기 전 현재 혼잡도와 추천 방문시간을 바로 확인할 수 있습니다.

---

### 2. 메뉴 주문 시연

동작:

1. 하단 탭에서 `메뉴주문` 선택
2. 덮밥 또는 백반 선택
3. 장바구니 하단 바 확인
4. `주문하기` 클릭

설명 예시:

> 학생은 키오스크 앞에 줄을 서지 않고 앱에서 메뉴를 선택한 뒤 주문할 수 있습니다.

---

### 3. 주문내역 / 픽업 관리 시연

동작:

1. 주문 후 자동으로 주문내역 화면 이동
2. 접수완료 상태 확인
3. 상태 시뮬레이션 버튼 클릭
4. 조리중, 픽업완료 상태 확인
5. QR 버튼 확인

설명 예시:

> 주문 후에는 음식 준비 상태를 단계별로 확인할 수 있어 불필요한 대기를 줄일 수 있습니다.

---

### 4. 식당정보 / AI 혼잡도 차트 시연

동작:

1. 하단 탭에서 `식당정보` 선택
2. 오늘 / 이번주 / 이번달 탭 확인
3. AI 혼잡도 차트 확인

설명 예시:

> 식당정보 화면에서는 운영시간 기준으로 시간대별 AI 혼잡도 예측을 확인할 수 있습니다. 과거 판매 데이터에서 11시부터 12시 사이에 주문 집중도가 높게 나타나는 패턴을 반영했습니다.

---

### 5. 관리자 대시보드 시연

동작:

1. 하단 탭에서 `관리자` 선택
2. 오늘 주문 수, 픽업 대기인원, 예상 매출 확인
3. 현재 주문 큐 확인
4. 메뉴 판매 순위 확인
5. AI 다음달 식수 예측 확인

설명 예시:

> 관리자 화면에서는 운영자가 실시간 주문 현황과 다음달 식수 예측을 확인할 수 있습니다. 이를 통해 식자재 준비량과 인력 배치를 더 효율적으로 조정할 수 있습니다.

---

### 6. 개발자 시연 모드

동작:

1. 관리자 탭 하단의 개발자 시연 모드 확인
2. `매우붐빔` 선택
3. 홈 화면으로 이동해 혼잡도 변화 확인
4. `여유` 선택 후 변화 확인
5. 마지막에는 `실시간`으로 복귀

설명 예시:

> 발표 시간이 실제 운영시간이 아닐 수도 있기 때문에, 시연 모드에서 여유부터 매우붐빔까지 다양한 AI 예측 상태를 즉시 보여줄 수 있도록 했습니다.

---

## 가장 임팩트 있는 시연 흐름

```text
홈 화면 → 메뉴 주문 → 주문내역 → 식당정보 → 관리자 → 개발자 시연 모드
```

이 순서가 가장 자연스럽고, 학생용 기능과 운영자용 기능을 모두 보여줄 수 있다.

---

## 시연 전 체크

- Chrome 또는 Edge에서 실행
- index.html 파일을 최신 파일로 열기
- 브라우저 캐시가 의심되면 새로고침(Ctrl + F5)
- 하단 탭이 모두 정상 동작하는지 확인
- 개발자 시연 모드에서 여유/혼잡/매우붐빔이 바뀌는지 확인
